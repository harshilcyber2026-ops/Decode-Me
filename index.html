⚑ Cryptography · Challenge #01

Decode Me

A suspicious string was intercepted in a network packet. The encoding is simple — but do you know what it is?

Target String

// intercepted payload · base64 encoded
RmxhZ1ZhdWx0e2I0czM2NF9pczNhc3l9

Scenario

During a routine network inspection, the analyst intercepted the string above embedded in an HTTP response header. It doesn't look like plain text — but it's not encrypted either.

Your task: Identify the encoding, decode the string, and submit the flag hidden inside it.

Clue: This encoding converts binary data to ASCII using exactly 64 printable characters. You see it every day on the web.

Quick Reference — Base64

📖 What is Base64?
Encodes binary data as ASCII text using A–Z, a–z, 0–9, +, /. Output length is always a multiple of 4 characters.
🔍 How to spot it?
May end with = or == padding. Only uses the 64-char alphabet. Output is ~33% longer than original input.
🐍 Python decode
import base64
base64.b64decode("RmxhZ...")
Returns a bytes object — call .decode().
💻 CLI decode
echo "RmxhZ..." | base64 -d
Works on Linux & macOS.
Windows: certutil -decode
